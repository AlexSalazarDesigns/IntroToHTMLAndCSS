What is HTML?

Hyper Text Markup Language

HTML is used for the placement or meaning of the content - header, paragraph, list versus its style (font, color, font size)

HTML Syntax highlights

Text is marked with a "tag" that gives it a purpose
  1. Meta Data
  2. Opening/closing tags
  3. Inline formatting
  4. Codecademy.com and W3Schools both have excellent tutorials

Basic HTML Best Practices

  1. Every open tag should have a matching closing tag
  2. Be case sensitive - maintain consistency
  3. Indent nested tags (such as when indenting outlines)
  4. Don't try to mash your code together. Use enough line breaks for readability and clarity.
  5. Validate often - save your work and see how it looks in-browser as you progress.

Debugging Tools

There are many tools you can use. Some are built into your text editor and some are separate. Here are some resources.

  • Google Developer tools
  • W3C HTML Validator

CSS Overview

Cascading Style Sheets (CSS)

CSS does the heaving lifting with regard to the page formatting.

CSS controls how elements look, similar to the way you use MS Word styles. You can specify font color, size, and other details that pertain to all elements in an HTML document that are the same type.

CSS Syntax Basics

The Rabbit Hole - A Preview of Future Lessons

Coding in raw CSS can get a little tricky, so other tools have been developed to make this process easier. These are vocabulary words you may hear for the first time tonight but we will spend more time on them later in the year.

  • Preprocessor - a utility program that aids in compatibility between the delivery method (CSS/HTML) and the receiving program (i.e. Google vs FireFox).
  • LESS and Sass are two such utilities.
  • They allow you to specify elements that are easier for a person to read and identify - @FacebookBlue instead of its actual hex color (i.e. #000000)
